## 1. 问题概述

本次 PRD 主要针对秒杀页、购物车页、商品页三个核心模块进行测试后发现的问题进行修复。登录/退出功能虽已实现，但与这些模块的交互存在多个严重问题，导致用户无法正常使用核心购物流程。

## 2. 测试结果汇总

| 模块 | 测试结果 | 严重程度 |
|------|----------|----------|
| 商品页 | 部分功能可用 | 中 |
| 秒杀页 | 功能不可用 | 高 |
| 购物车页 | 功能完全不可用 | 高 |

---

## 3. 详细问题列表

### 3.1 商品页问题

#### 问题 1.1: 商品库存显示异常
- **问题描述**: 商品列表中显示库存为 0，但数据库中应该有库存数据
- **当前表现**: 页面显示 "库存: 0"
- **预期表现**: 应显示真实库存数量

#### 问题 1.2: 商品数据过少
- **问题描述**: 目前仅有 1 条测试商品数据
- **当前表现**: 只显示 1 个 "测试商品"
- **预期表现**: 应有完整的商品数据供用户浏览选购

---

### 3.2 秒杀活动页问题 (严重)

#### 问题 2.1: 秒杀 API 路由配置错误
- **问题描述**: 点击"立即抢购"按钮后，请求返回 HTML 页面而不是 JSON 响应
- **当前表现**:
  ```
  Console Error: SyntaxError: Unexpected token '<', "<!DOCTYPE "... is not valid JSON
  ```
- **影响**: 用户完全无法参与秒杀活动
- **请求路径**: `GET /api/v1/activity/token?activity_id=1`
- **根本原因**: 该 API 路径未在网关或后端正确配置，返回了前端 HTML 页面

#### 问题 2.2: 秒杀活动数据来源不明确
- **问题描述**: 秒杀活动页面显示的商品数据（iPhone 15 Pro、MacBook Pro 等）来源不清晰
- **当前表现**: 页面显示了 4 个秒杀商品，但商品列表页只有 1 个商品
- **预期表现**: 秒杀商品应与商品库关联

---

### 3.3 购物车页问题 (严重)

#### 问题 3.1: 购物车列表 API SQL 语法错误
- **问题描述**: 获取购物车列表时返回 500 错误
- **错误信息**:
  ```
  rpc error: code = Unknown desc = pq: syntax error at or near "," at column 12 (42601)
  ```
- **请求路径**: `GET /douyin/cart/list`
- **根本原因**: Cart Model 使用 MySQL 占位符 `?`，但数据库是 PostgreSQL，应使用 `$1, $2` 占位符

#### 问题 3.2: 添加购物车 API SQL 语法错误
- **问题描述**: 点击"加入购物车"按钮时返回 500 错误
- **错误信息**: 与问题 3.1 相同
- **请求路径**: `POST /douyin/cart/add`
- **根本原因**: 同问题 3.1

## 4. 修复方案

### 4.1 商品页修复

| 问题编号 | 修复方案 | 负责端 |
|----------|----------|--------|
| 1.1 | 检查商品 API 返回的 stock 字段，确保从数据库正确读取 | 后端 |
| 1.2 | 补充商品测试数据，或确保商品数据正确导入 | 后端/数据库 |

### 4.2 秒杀活动页修复

| 问题编号 | 修复方案 | 负责端 |
|----------|----------|--------|
| 2.1 | 在网关(gateway)中正确配置 `/api/v1/activity/token` 路由，指向 activity 服务 | 后端 |
| 2.2 | 确认秒杀活动商品数据来源，关联商品表 | 后端 |

**技术要点**:
- 检查 `services/gateway/etc/gateway.yaml` 中的路由配置
- 确认 activity 服务的启动状态和端口配置

### 4.3 购物车页修复

| 问题编号 | 修复方案 | 负责端 |
|----------|----------|--------|
| 3.1 | 修改 `dal/model/cart/cartsmodel_gen.go` 中的 SQL 语句，将 `?` 占位符改为 PostgreSQL 的 `$1, $2` 格式 | 后端 |
| 3.2 | 同问题 3.1 | 后端 |

**技术要点**:
- 需要检查所有使用 `?` 占位符的 SQL 语句
- 或者配置 sqlx 使用 PostgreSQL 方言
- 需要全面检查其他 Model 是否存在相同问题
