# 后端实现要求文档（PRD_backend.md）
## 一、功能概述
实现基于长短令牌的分布式用户登录态管理系统，支持高并发场景下的用户身份验证、自动续期、强制下线等功能。

## 二、核心模块实现要求

### 1. 登录服务模块
#### 1.1 长令牌生成逻辑
- 用户登录成功后，生成全局唯一的 `SessionID`（建议使用UUID v4）
- 在Redis中存储Session数据，Key格式：`user:session:{session_id}`，Value为JSON格式的用户信息：
  ```json
  {
    "user_id": 123,
    "device_id": "设备ID",
    "login_ip": "登录IP",
    "login_time": 1234567890,
    "expire_time": 1234567890 + 30*24*3600
  }
  ```
- Session过期时间设置为30天（2592000秒）
- 长令牌格式：`{SessionID}.{数字签名}`，数字签名使用HMAC-SHA256算法，密钥为服务端私钥

#### 1.2 短令牌生成逻辑
- 短令牌组成：`{user_id}.{device_id}.{expire_time}.{数字签名}`
  - `expire_time` 为当前时间+1天（86400秒）的时间戳
  - 数字签名使用与长令牌相同的HMAC-SHA256算法
- 短令牌不需要存储在Redis中，采用自包含验证方式

#### 1.3 令牌返回
- 登录成功后，将 `long_token` 和 `short_token` 返回给客户端
- 支持通过Cookie和响应体两种方式返回

### 2. 网关鉴权中间件
#### 2.1 短令牌验证流程（优先验证）
1. 从请求头中获取 `short_token`
2. 拆分令牌，验证数字签名是否合法
3. 验证短令牌是否过期（当前时间 > expire_time）
4. 如果签名合法且未过期：验证通过，将用户信息注入请求上下文，继续处理请求
5. 如果签名不合法：返回401错误，错误信息 `令牌非法`，要求重新登录
6. 如果令牌已过期：进入长令牌验证流程

#### 2.2 长令牌验证流程（短令牌过期时执行）
1. 从请求头中获取 `long_token`
2. 拆分令牌，验证数字签名是否合法
3. 如果签名不合法：返回401错误，错误信息 `Session已过期`，要求重新登录
4. 如果签名合法：根据SessionID查询Redis中的Session数据
5. 如果Session不存在：返回401错误，错误信息 `Session不存在`，要求重新登录
6. 如果Session存在且有效：
   - 生成新的短令牌（有效期重新计算为1天）
   - 将新的短令牌放入响应头 `Short-Token-Refresh` 中返回给客户端
   - 将用户信息注入请求上下文，继续处理请求

### 3. 强制下线功能实现
- 当需要强制某用户下线时，直接删除Redis中对应用户的所有Session数据
- 短令牌过期时间决定了强制下线的生效时间（默认1天内生效）
- 如需立即生效，可以将用户ID加入Redis黑名单，中间件验证时先检查黑名单

### 4. 登出接口实现
- 从请求头中获取长令牌，解析出SessionID
- 删除Redis中对应的Session数据
- 返回登出成功响应

### 5. 刷新令牌接口实现
- 验证长令牌合法性
- 验证Session存在且有效
- 生成新的短令牌返回给客户端
- 可选：支持延长Session有效期

## 三、安全要求
1. 签名私钥需要严格保密，定期轮换
2. Redis中Session数据需要加密存储，防止数据泄露
3. 接口需要限流，防止暴力破解令牌
4. 记录所有登录、登出、令牌刷新的操作日志，便于审计
5. 支持多端登录管控：同一账号最多同时登录N个设备，超出时自动踢掉最早登录的设备
6. 异地登录、异常设备登录时需要发送提醒给用户

## 四、性能要求
1. 令牌验证逻辑需要在网关层完成，避免请求打到后端服务
2. 单次令牌验证耗时要求 < 1ms
3. Redis Session查询需要支持高并发，建议使用集群部署
4. 支持横向扩展，无状态设计

## 五、测试用例
| 测试场景 | 预期结果 |
|---------|---------|
| 合法长短令牌请求 | 验证通过，正常处理请求 |
| 短令牌过期，长令牌有效 | 自动生成新的短令牌返回，请求正常处理 |
| 短令牌签名被篡改 | 返回401，拒绝请求 |
| 长令牌签名被篡改 | 返回401，拒绝请求 |
| Session已被删除 | 返回401，拒绝请求 |
| 强制用户下线后请求 | 短令牌过期后返回401，要求重新登录 |
| 并发大量请求 | 服务稳定，无性能瓶颈 |
| 多设备同时登录同一账号 | 各设备登录态独立，互不影响 |

## 六、配置参数
| 参数 | 建议值 | 说明 |
|------|--------|------|
| 长令牌有效期 | 30天 | 用户登录态最大有效期 |
| 短令牌有效期 | 1天 | 短令牌过期时间，影响强制下线生效时间和Redis访问压力 |
| 签名算法 | HMAC-SHA256 | 数字签名算法 |
| Session Key前缀 | `user:session:` | Redis中Session存储的Key前缀 |
| 黑名单Key前缀 | `user:blacklist:` | 强制下线用户黑名单Key前缀 |
